# Выпуск goplay.mobi в App Store и Google Play

Аудит готовности к ревью сторов и чек-лист выката в прод. Стадия: клиент готов
(MVP), осталось внешнее (аккаунты разработчика, хостинг юр-страниц, ключи, сборка).

## 1. Что уже сделано под ревью (в коде)

- ✅ **Удаление аккаунта в приложении** (Apple Guideline 5.1.1(v), Google Play
  «Account deletion»): Профиль → «Удалить аккаунт» с подтверждением →
  `DELETE /api/mobi/account` (удаляет пользователя, при FK-блокировке — стирает PII).
- ✅ **Демо-доступ для ревьюеров без реального SMS**: если на бэке задан
  `DEMO_PHONE` + `DEMO_OTP`, рецензент входит по этому номеру с фиксированным кодом
  (SMS не шлётся). На обычных номерах поведение не меняется. Это снимает риск
  отказа «не смогли войти» (Apple 2.1), пока SMS-провайдер не подключён.
- ✅ **Юр-ссылки в приложении**: Профиль → Политика конфиденциальности / Условия
  (открываются в браузере). URL: `https://goplay.tj/privacy`, `/terms` — **страницы
  нужно опубликовать** (см. §3).
- ✅ **Разрешения с понятными описаниями**: камера (QR), геолокация (карта),
  уведомления. Покупки кошелька открываются во внешнем браузере (Apple 3.1.1 —
  оплата физической услуги в клубе).
- ✅ **Иконка** 1024×1024 RGB без альфа-канала (Apple это требует), сплэш 2732².
- ✅ **Просмотр без входа**: каталог клубов и карта доступны без логина; вход нужен
  только для личного (брони/кошелёк/скан) — не нарушает 5.1.1(i).
- ✅ Версия `1.0.0`, экспорт-комплаенс выключен (`ITSAppUsesNonExemptEncryption=false`),
  bundleId/package `tj.goplay.mobi`.

## 2. Риски ревью и как закрыты

| Риск | Гайдлайн | Статус |
|---|---|---|
| Нет удаления аккаунта | Apple 5.1.1(v) | ✅ добавлено |
| Рецензент не может войти (нет SMS) | Apple 2.1 | ✅ демо-номер |
| Платёж за цифру мимо IAP | Apple 3.1.1 | ⚠️ трактуем как оплату **физической** услуги (игровое время в клубе) — внешний браузер допустим. В заметках ревью объяснить. |
| Нет политики конфиденциальности | Apple 5.1.1 / Google | ⚠️ ссылки есть, **страницы надо опубликовать** |
| Принудительная регистрация | Apple 5.1.1(i) | ✅ просмотр открыт |
| Push без запроса | — | ✅ запрос разрешения штатный |

## 3. Что нужно от вас для выката (внешнее)

### Аккаунты
- [ ] **Apple Developer Program** ($99/год) → создать приложение в App Store Connect,
      получить `ascAppId`, `appleTeamId`, Apple ID — вписать в `eas.json → submit.production.ios`.
- [ ] **Google Play Developer** ($25 разово) → создать приложение, скачать
      **service-account JSON**, положить как `google-play-service-account.json`.

### Юридические страницы (обязательно для обоих сторов)
- [ ] Опубликовать **Политику конфиденциальности** на `https://goplay.tj/privacy`.
- [ ] Опубликовать **Условия использования** на `https://goplay.tj/terms`.
- [ ] Для Google Play Data Safety — указать способ запроса удаления данных
      (в приложении есть, плюс можно дать веб-URL).

### Ключи / окружение (бэкенд и сборка)
- [ ] **`GOOGLE_MAPS_API_KEY`** (Android-карта) → `eas secret` или env прод-профиля.
- [ ] На бэкенде задать **`DEMO_PHONE` + `DEMO_OTP`** (для ревью), передать их в
      заметках рецензенту.
- [ ] (Позже, не блокирует ревью с демо) SMS-провайдер: `SMS_PROVIDER=oson` + креды.

### Материалы для листинга стора
- [ ] **Скриншоты**: iPhone 6.7" и 5.5" (обяз.), iPad если `supportsTablet`. Можно
      взять из готовых макетов `/screens`.
- [ ] Android: feature graphic 1024×500, иконка 512×512, скриншоты.
- [ ] Тексты: название, краткое/полное описание, ключевые слова, support URL,
      возрастной рейтинг (анкета), категория.
- [ ] **App Privacy / Data Safety**: задекларировать сбор (телефон — аккаунт;
      геолокация — поиск клубов; без трекинга между приложениями).

## 4. Команды сборки и сабмита

```bash
# вход и привязка проекта
eas login
eas build:configure

# секреты
eas secret:create --name GOOGLE_MAPS_API_KEY --value "<ключ>"

# прод-сборки
eas build -p ios --profile production
eas build -p android --profile production

# отправка в сторы (после заполнения eas.json → submit.production)
eas submit -p ios --profile production --latest
eas submit -p android --profile production --latest
```

## 5. Заметка рецензенту (вставить в App Review / Play review)

> Вход по номеру телефона. Для проверки используйте демо-номер: **<DEMO_PHONE>**,
> код подтверждения: **<DEMO_OTP>** (реальная SMS не отправляется). После ввода
> кода задайте любой пароль (≥6 символов) — попадёте в аккаунт.
> Пополнение кошелька — это предоплата игрового времени в физическом
> компьютерном клубе (услуга оказывается офлайн), поэтому оплата идёт через
> внешний платёжный сервис, а не через IAP.

## 6. Перед каждым релизом

- [ ] Поднять `version` (и при ручном режиме — build/versionCode; сейчас
      `autoIncrement` в production-профиле).
- [ ] `npx tsc --noEmit` чисто.
- [ ] Прогнать вход (демо-номер), бронь, кошелёк, скан, удаление аккаунта.
