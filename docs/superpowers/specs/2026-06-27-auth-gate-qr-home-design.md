# Дизайн: гейт авторизации, локальная эмуляция входа, QR-сканер как главная

Дата: 2026-06-27
Статус: утверждён к реализации

## Цель

1. Без входа приложение сразу показывает экран входа/регистрации (полный гейт).
2. Залогиненный клиент первым видит QR-сканер — основная страница, по центру
   таб-бара, выделенная цветом.
3. При регистрации клиенту явно сообщается, какими данными он входит
   (номер телефона + единый пароль для всех клубов сети Goplay).
4. Пока нет SMS-кредов — вход полностью эмулируется в приложении: ОТП = `123456`.
5. Модуль «Клубы» (список + карта + детальная клуба) убирается.

## Решения (зафиксированы с заказчиком)

- Полный гейт: гостевого режима нет.
- ОТП — статичный `123456`, проверяется **локально в приложении** (бэкенд входа
  пока не используется). Всё за флагом `AUTH_STUB` для лёгкого отката.
- ОТП используется при регистрации и при сбросе пароля («Забыли пароль?»);
  обычный повторный вход — по номеру + паролю, без ОТП.
- Таб-бар: 3 вкладки — `Брони · [СКАН] · Кошелёк`. «Скан» по центру, приподнят,
  цвет `colors.primary`. «Профиль» — иконкой в шапке (стек-роут с «назад»).

## Архитектура

### 1. Локальная эмуляция входа (`AUTH_STUB`)

- `src/config.ts`: `export const AUTH_STUB = true;` + `STUB_OTP = '123456'`.
- Новый `src/auth/stubAuth.ts` — эмулирует `authApi` поверх AsyncStorage:
  - аккаунты: `{ [phone]: { password, user } }` в ключе `goplay.stub.accounts`;
  - `stubStart` → `{ registered }`; `stubRequestCode` → no-op `{ ok, resendIn }`;
  - `stubVerifyCode` → принимает только `123456`, иначе `ApiError`;
  - `stubSetPassword` → создаёт/обновляет аккаунт, возвращает `{ token, user }`;
  - `stubLogin` → проверяет номер+пароль локально;
  - `stubGetUser(token)` / `stubDeleteAccount()` — для сессии и удаления;
  - токен формата `stub:<phone>`, user — минимальный (`id, phone, username`).
- `src/api/auth.ts`: каждый метод `authApi` при `AUTH_STUB` делегирует в stub,
  иначе — прежний вызов `mobi.*`. `app/login.tsx` не меняет вызовов.
- `src/api/client.ts`: добавить `getAuthToken()` (нужен stub-удалению аккаунта).
- `src/auth/AuthContext.tsx`: в `applyToken` при `AUTH_STUB` брать пользователя
  из `stubGetUser(token)` вместо `clubApi.me()`.

### 2. Полный гейт авторизации

- `app/_layout.tsx`: компонент `RootNav` читает `useAuth()`:
  - `loading` → полноэкранный спиннер;
  - через `Stack.Protected guard={!!user}` показываются `(tabs)` и `profile`;
  - через `Stack.Protected guard={!user}` показывается `login`.
- `login` перестаёт быть модалкой (`headerShown: false`, полноэкранный).
- `app/login.tsx`: убрать `router.back()` после `signInWithToken` (гейт сам
  перерисует на вкладки). Ссылка «Забыли пароль?» запускает сброс через ОТП
  (код → новый пароль). Тексты регистрации обновить (см. п.5). Поле кода
  предзаполнять `123456` с подсказкой.

### 3. Стартовый экран = «Скан», кастомный таб-бар (3 вкладки)

- `app/(tabs)/_layout.tsx`:
  - `export const unstable_settings = { initialRouteName: 'scan' }`;
  - вкладки только `bookings`, `scan`, `wallet` (в этом порядке);
  - `scan` — кастомный `tabBarButton`: приподнятый круг `colors.primary`,
    иконка `qr-code` белая, подпись «Скан»;
  - `screenOptions.headerRight` — иконка «Профиль» → `router.push('/profile')`.

### 4. Профиль — стек-роут

- `app/(tabs)/profile.tsx` → `app/profile.tsx` (открывается поверх вкладок,
  нативная кнопка «назад»). Регистрируется в `RootNav` внутри guard-зоны входа.
- Логаут и удаление аккаунта работают через `AuthContext`/stub; после выхода
  гейт показывает экран входа.

### 5. Удаление модуля «Клубы»

- Удалить `app/(tabs)/index.tsx`, `app/(tabs)/map.tsx`, `app/club/[slug].tsx`
  (ссылки на `/club/[slug]` были только из этих файлов).
- Снять обёртку `RequireAuth` со `scan/wallet/bookings`; удалить
  `src/components/RequireAuth.tsx` (гейт делает её лишней).

### 6. Тексты регистрации

- Шаг «Придумайте пароль»: подзаголовок
  «Это пароль для входа во все клубы сети Goplay. Логин — ваш номер телефона.»
- На шаге ввода кода — подсказка «Тестовый код: 123456».

## Поток данных

`AuthContext` хранит `user/token/loading`. При `AUTH_STUB` источник истины —
локальное хранилище stub; реальные эндпоинты `/api/mobi/auth/phone/*` не
вызываются. Экраны вкладок получают гарантированно ненулевого `user`.

## Вне объёма

- Бэкенд (единый пароль сети, реальный SMS-провайдер) — отдельная задача;
  при готовности `AUTH_STUB = false` возвращает реальные вызовы.
- Брони/Кошелёк/Скан продолжают ходить на бэкенд и без него показывают пустые
  состояния / ошибку разблокировки — это ожидаемо.

## Проверка

- `npx tsc --noEmit` — чисто.
- Сборка бандла Metro (HTTP 200) для ios и android.
- Логический прогон: старт без сессии → экран входа; регистрация с `123456`
  → задаётся пароль → открыт «Скан» по центру; повторный запуск → вход по
  паролю; логаут → снова экран входа.
